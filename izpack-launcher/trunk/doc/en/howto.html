1. Introduction

Izpack-launcher is a native Executable, that searches for, installs if not found and starts a Java Runtime Environment (JRE) together with a customers "IzPack Java - setup.jar". If Java is already installed in the matched version, the izpack-launcher (autorun) setup.exe just starts this Izpack Setup immediatelly, and it seems (in the scope of the customer or user) that the setup.exe performs the entire setup application. There can also be made a self-extracting Version, but this uses an another advanced technique and is not part of this document (It is described by Julien in a separate "screen-cam" demonstration). Finally, IzPack-Launcher is really a very neat tool, that supports IzPack greatful, but even on Linux, you can also write a more or less large Shellscript, that covers the same tasks. And last Imho: Izpack-Launcher is not the name that describes this Executable exactly. JRE_FindAndStartWithAny(Gui)Jar.exe would be a more precise name. ;-)

3. Startup 

Izpack launcher is build and based on WxWindows/wxWidgets, a native and platform independend GUI-Development library similar to Qt and Gtk. But it is more free as Qt even for win32 binaries at the time of writing (until the Qt4 release). That is why you will need (additionally to a C++ Compiler) the WxWindow/wxWidgets distribution. I will describe how to get and prebuild this GUI-library in the Prerequisites and the Preparation Section.

To use Izpack-Launcher, you also need an executable "Izpack-Setup Jar-File". But you don't need an own setup of your software. To immediate start, you can also alternatively use IzPack's Setup-Jar. Just download it from http://www.izforge.com/izpack, if needed.

For more information, how to use, embedd and build a Izpack Setup with the IzPack-Launcher, read both, the manual, which comes with IzPack and the other, which comes with the IzPack Launcher. A quickstart tutorial as pdf is also provided and is recommended for beginners. There is another "howto compile" in the WIKI, which describes the compile and build based on MSVC++6.0 and Open Watcom. Feel free, to decide your appropriate way to build.

4. Prerequisites 

At first, you need a C++ Compiler. The easiest way is to download and install DevCPP from http://www.bloodshed.net.
This is a complete IDE but it comes with all tools needed to compile wxWindows/wxWidgets and izpack-launcher. The current release of DevCPP is 4.9.9.2. Unfortunatelly the make utility of this suite has a bug, so you have to edit/copy 3 Files manually. I will describe this at the preparation chapter. Note: if this bug was fixed in a future release, you don't need no longer do so.

After installation, you have to add the "bin" - folder of DevCPP to your PATH-Environmentvariable. i.e. set PATH=%PATH%;C:\devcpp\bin

5. Getting Sources 

Next thing to build izpack-laucher, you have to do, is to checkout the sources from the svn-tree, if you don't have the sources already. As registered izpack developer, you can checkout using:
svn checkout svn+ssh://developername@svn.berlios.de/svnroot/repos/izpack/trunk
while as anonymous user you have to use:
svn checkout svn://svn.berlios.de/izpack/trunk

Then, you have to download the wxWindows/wxWidgets sources for your target platform. In this document, I will describe the compilation on the windows platform, so you have to download wxMSW-2.4.2-setup.zip from http://www.sourceforge.net . I use this old wxWidgets-release explicitly, because with the newer release, it does not work out of the box. (This is maybe a todo for us - at least we have to adapt the makefiles.)

6. Installation

Once you have download all needed things, you are ready to install these. I have setup DevCPP with the default options and the defualt installpath: c:\dev-cpp. wxWindows was also installed typically, except I choosed another install path: c:\wxw242.

7. Preparation 

You have to read the documentation, howto build the wxWindows library, but I have extracted the requiered steps to compile with DevCpp.

A) In order to compile the wxWindows/wxWidget Libraries, you have to set some environment variables. You can set these globally with Settings --> Control panel -> System ->Advanced -> Environment Variables or only in the current Console:

i) Adjust the environment variable WXWIN to your wxWindows/wxWidgets installation-  base path: 

  > set WXWIN=C:\wxw242

ii) Don't forget to add the "bin" subdirectory of DevCPP to your PATH environment variable:

   > set PATH=%PATH%;C:\dev-cpp\bin

iii) Now open a new console (cmd.exe), change to the wxWindows basedir typing:

> cd /d C:\wxw242\src\msw 

iv) and enter:

>  make -f makefile.g95 clean

 to cleanup previous created binaries. This is not required, for the first time.

v) now enter:

> make -f makefile.g95 FINAL=1

according to the wx compile instruction fow mingw. 

NOTE: Unfortunatelly, there si a tiny bug in the make - Tool or in the called copy-tool. (The destination option is not recognized correctly), 
So you will get a Errormesage such as:

mkdir ..\..\lib\msw
mkdir ..\..\lib\msw\wx
cp ../../include/wx/msw/setup.h ../../lib/msw/wx/setup.h
process_begin: CreateProcess((null), cp ../../include/wx/msw/setup.h ../../lib/m
sw/wx/setup.h, ...) failed.
make (e=2): The system cannot find the file specified.
make: *** [../../lib/msw/wx/setup.h] Error 2

or:

cp ../common/dosyacc.c ../common/y_tab.c
process_begin: CreateProcess((null), cp ../common/dosyacc.c ../common/y_tab.c, .
..) failed.
make (e=2): The system cannot find the file specified.
make: *** [../../src/common/y_tab.c] Error 2


However there is a simple Workaround by doing the three following easy things ;-) : 

Just copy the file manually: Enter in the case above:
Note: Use backslash instead of slash.

> copy ..\..\include\wx\msw\setup.h ..\..\lib\msw\wx\setup.h
comment out line 546: ( $(COPY) $(WXDIR)/include/wx/msw/setup.h $(subst $(BACKSLASH),/,$@) ) in the makefile.g95 and continue the build, by reenter  
> make -f makefile.g95 FINAL=1

> copy ..\common\dosyacc.c ..\common\y_tab.c

and continue the build, by enter  

> make -f makefile.g95 FINAL=1

> copy ..\common\doslex.c ..\common\lex_yy.c

again and finish the wxWin-build, by (last) re-enter  

> make -f makefile.g95 FINAL=1

The wxwindows libs and all other wxw stuff should now compiled successfully.

OK. You are now able to compile the Izpack-Launcher Exe.

8. Perfom complete compilation of the Izpack-Launcher - Distribution

( First ensure, that the WXWIN-Environmentvariable is set. Otherwise you get an error like: 

Makefile.mingw:10: /src/makeprog.g95: No such file or directory
make: *** No rule to make target `/src/makeprog.g95'.  Stop.)

If all is setup correctly, you can simply compile by Enter the following command:

C:\izpacklauncher>make -f Makefile.mingw install
